## Before you start

You'll need a few things installed:

- [Node.js](https://nodejs.org/) v18 or higher
- MongoDB — either [running locally](https://www.mongodb.com/try/download/community) or a free cluster on [MongoDB Atlas](https://www.mongodb.com/atlas)
- npm (comes with Node)

## 1. Clone the repo

```bash
git clone https://github.com/your-org/skills-api.git
cd skills-api
```

## 2. Install dependencies

```bash
npm install
```

## 3. Set up your environment

Create a `.env` file in the root of the project. Here's what it should look like:

```bash .env
PORT=5000
MONGO_URI=mongodb://127.0.0.1:27017/skills_db
ADMIN_API_KEY=supersecretadminkey123
```

Change `ADMIN_API_KEY` to something actually secret before you deploy anywhere.

<Warning>
  Don't commit your `.env` file. If it's not already in your `.gitignore`, add it now.
</Warning>

## 4. Start the server

```bash
npm start
```

If everything's wired up correctly, you'll see:

```
MongoDB connected
Server running on port 5000
```

If MongoDB isn't running you'll get a connection error here — start MongoDB first, then try again.

## 5. Make your first request

Let's search for a skill to confirm the API is alive:

```bash
curl "http://localhost:5000/api/skills/search?keyword=docker&page=1"
```

You should get something back like this:

```json
{
  "data": [
    {
      "skill_id": "sk-001",
      "Skill": "Docker",
      "sub_skill_id": "sub-001",
      "master_skill_id": "ms-001"
    }
  ],
  "page": 1,
  "limit": 10,
  "count": 1
}
```

If your database is empty, `data` will be an empty array — that's fine, it means the API is working.

## Project structure

Here's a quick map of where things live:

```
.
├── app.js                          # Entry point — routes, middleware, DB connection
├── middleware/
│   ├── requireApiKey.js            # Blocks unauthenticated admin requests
│   ├── validate.js                 # Runs Joi schemas against request bodies
│   └── detectScraping.js           # Watches for suspiciously high request rates
├── models/
│   ├── MasterSkill.js
│   ├── SubSkill.js
│   ├── Skill.js
│   └── SelectedSkill.js
├── routes/
│   ├── skills.routes.js
│   ├── selectedSkills.routes.js
│   └── admin.routes.js
├── validations/
│   └── selectedSkill.validation.js
└── swagger.js                      # Auto-generates /api-docs from JSDoc comments
```
