## Install dependencies

```bash
npm install
```

Key packages used:

| Package | Purpose |
|---|---|
| `express` | HTTP server and routing |
| `mongoose` | MongoDB ODM |
| `joi` | Request body validation |
| `uuid` | Generating `selected_skill_id` values |
| `express-rate-limit` | Per-route and global rate limiting |
| `swagger-jsdoc` + `swagger-ui-express` | Auto-generated API docs at `/api-docs` |
| `dotenv` | Loading `.env` variables |
| `helmet` *(recommended)* | Secure HTTP headers |
| `cors` *(recommended)* | Cross-origin request control |

## Environment Variables

| Variable | Required | Description |
|---|---|---|
| `MONGO_URI` | ✅ | MongoDB connection string |
| `ADMIN_API_KEY` | ✅ | Secret key for `/api/admin/*` routes |
| `PORT` | ❌ | HTTP port (default: `5000`) |

## Running the server

```bash
# Development (with nodemon auto-restart)
npm run dev

# Production
npm start
```

## Swagger UI

While the server is running, visit:

```
http://localhost:5000/api-docs
```

All routes are documented automatically from JSDoc comments in the route files.

## Rate Limits

| Scope | Limit | Window |
|---|---|---|
| Global — all routes | 100 requests / IP | 15 minutes |
| `/api/skills/search` | 15 requests / IP | 1 minute |
| Scraping detection | 30 requests / IP | 5 minutes (rolling) |

## Adding a New Route

1. Create your route file in `routes/`
2. Register it in `app.js` with `app.use("/api/your-route", require("./routes/your-route"))`
3. Add Joi validation schema in `validations/` if the route accepts a body
4. Wire validation using the `validate` middleware:

```js
const validate = require("../middleware/validate");
const { yourSchema } = require("../validations/your.validation");

router.post("/", validate(yourSchema), async (req, res) => { ... });
```

5. Add JSDoc comments above the route for Swagger to pick up automatically
